## New CLI surface

### Surgical-write commands (Brief 05)

For agents managing vaults programmatically without full working copies:

- **`sgit write <path> <vault>`** — write a single file to vault HEAD
  in one command. Supports `--push`, `--message`, stdin / `--file`,
  `--json` output.
- **`sgit cat <path> --id`** — print the object-id alongside content.
- **`sgit cat <path> --json`** — machine-readable output (path,
  object-id, size, content).
- **`sgit ls --ids`** — list with object-ids.
- **`sgit ls --json`** — machine-readable directory listing.
- **`sgit clone --read-key <hex>:<vault-id>`** — clone in read-only
  mode using a share key (no vault key required).
- **`sgit derive-keys`** now accepts a `read_key:vault_id` format and
  returns the derivable subset (read-only mode).
- **`sgit info`** now reports write-key availability:
  `Write key: ✓ available` (full clone) or `Write key: ✗ not available
  (re-clone with full vault key to write)`.

Read-only clones now **fail closed** on write attempts with a clear
error: `"This vault was cloned read-only. To write, re-clone with the
full vault key."`

## Security hardening (8 AppSec findings closed)

- **F02 / F07 / F11 — File-mode hygiene.** Every write under
  `.sg_vault/local/` (vault_key, config.json, clone_mode.json,
  pending_registration.json, push_state.json, merge_state.json, tokens,
  share history, debug flag) is now `chmod 0600`. Single helper
  (`Vault__Storage.chmod_local_file`) — applied at every site.
- **F02 — Secure unlink for rekey.** `rekey_wipe` now overwrites file
  contents with zero bytes + `fsync` before `os.unlink`, via new
  `Vault__Storage.secure_unlink` and `secure_rmtree`. Closes the
  rekey key-material disk-residency window. (Note: SSD TRIM may
  independently reallocate blocks; documented as residual risk.)
- **F03 — PBKDF2 LRU cache clearing.** `Vault__Crypto.clear_kdf_cache()`
  added; called at passphrase boundaries (after `delete_vault`, after
  `rekey_wipe`, at every `probe_token` exit path). Closes the
  long-running-agent passphrase residency window. Closes mutation
  matrix row M5.
- **F08 — `write_file` read-only guard.** Defensive guard at
  `Vault__Sync__Commit.write_file` raises `Vault__Read_Only_Error`
  before any I/O when `write_key` is absent. Prevents silent demotion
  of a read-only clone via corrupt `clone_mode.json`. New typed
  exceptions in `sgit_ai/sync/Vault__Errors.py`.
- **`clone_mode.json` fail-closed.** Corrupt or malformed
  `clone_mode.json` now raises `Vault__Clone_Mode_Corrupt_Error`
  instead of silently falling back to full-mode treatment.

## Bug fixes

- **`delete_on_remote` left stale `push_state.json`** (Architect
  finding 06). Next push to a re-created vault of the same id would
  skip Phase-A blob upload and produce dangling references. Now
  cleared inside `delete_on_remote`. End-to-end test in
  `test_Vault__Sync__Delete_Push_State.py`.
- **`reset()` NameError** (`Vault__Sync__Pull.py`). `vault_commit`
  was referenced before definition (had a `# noqa: F821` workaround).
  Fixed by constructing `Vault__Commit` before first use.
- **`destroy` missing access token** (`Vault__API.py`). The
  `x-sgraph-access-token` header was absent from the DELETE request.
  Added.
- **Dead code in `Vault__Sync__Status.py`** — two `else` branches
  mathematically proven unreachable via BFS-walk analysis, removed.
  Brought `Vault__Sync__Status` to 100% coverage.

## Internal architecture

### `Vault__Sync.py` split

`sgit_ai/sync/Vault__Sync.py` was 3,032 LOC. After v0.12.0, it's a
**258-line facade**. Twelve sub-classes carry the implementation:

```
Vault__Sync__Base.py       Vault__Sync__Pull.py        Vault__Sync__Branch_Ops.py
Vault__Sync__Commit.py     Vault__Sync__Push.py        Vault__Sync__GC_Ops.py
Vault__Sync__Status.py     Vault__Sync__Clone.py       Vault__Sync__Lifecycle.py
Vault__Sync__Admin.py      Vault__Sync__Sparse.py      Vault__Sync__Fsck.py
```

**Public API is unchanged.** `Vault__Sync` delegates each method to
the appropriate sub-class. Existing scripts and integrations keep
working.

### Type_Safe schemas for state files

Three on-disk JSON files now have proper Type_Safe schemas:
`Schema__Push_State`, `Schema__Clone_Mode`, `Schema__Local_Config`.
Round-trip invariant tests + extra-field rejection close mutation
gap M8 structurally.

### `Vault__Sub_Tree` deduplication

Common dir-walk and encrypt-or-reuse logic extracted into shared
methods (B22 E1 + E2). Sets up the v0.12.x perf work for further
deduplication.

## Test infrastructure milestone

- **Shared fixtures** under `tests/conftest.py` and
  `tests/_helpers/vault_test_env.py`:
  - `known_test_keys` — pre-derived keys for 5 canonical vault keys.
  - `precomputed_encrypted_blobs` — pre-encrypted blobs (small / medium / large).
  - `two_clones_pushed` — Alice init+push, Bob clone snapshot.
  - `vault_with_N_commits` — parameterised N={1, 5, 20} commit history.
  - `vault_with_pending_changes` — dirty working copy.
  - `vault_with_branches` — two-branch diverged vault.
  - `read_only_clone` — read-only clone snapshot.
- **`Vault__Test_Env`** — class-level snapshot + `restore()` for
  per-test isolation (~3 ms restore vs ~400 ms full vault init).
- **CI parallelisation**: `pytest-xdist` two-pass shape (`-n auto`
  parallel pass + serial `no_parallel` pass).
- **Mutation-test orchestrator** at `tests/mutation/run_mutations.py`
  — uses `git worktree` per mutation for CI safety; replaces the
  manual `git stash` approach.
