## Sub-finding 2.2 — `monkeypatch` files (519 lines, 19 files)

`monkeypatch.setattr` is being used as a substitute for `@patch`. From
the project's no-mocks rule: monkeypatch on production code (not on
environment variables) is functionally identical to a `@patch` and is a
violation of the same rule.

Sample (from `tests/unit/cli/test_CLI__Branch.py:157–172`):

```python
def test_branch_new_runtime_error_exits(self, monkeypatch, capsys):
    ...
    monkeypatch.setattr(Vault__Branch_Switch, 'branch_new',
                        lambda *a, **k: (_ for _ in ()).throw(RuntimeError('boom')))
```

This is a homemade replacement for `@patch.object(Vault__Branch_Switch,
'branch_new')` and counts as a mock for the no-mocks rule.

19 files using `monkeypatch` — the full list is generated by
`grep -rln 'monkeypatch' tests/`. Most are test files **modified**
during the sprint (carryover) but none of the 7 newly-added sprint
test files use `monkeypatch`.

## Sub-finding 2.3 — Homemade mocks in **newly-added** sprint tests

`tests/unit/cli/test_CLI__Vault__Write.py` uses an inline `class
FakeArgs` four times, lines 98, 117, 172, 224. Each `FakeArgs` is a
plain Python class with attribute defaults that pretends to be the
output of `argparse.Namespace`. This is a stub. It is **newly
introduced** by this sprint (file added in commit `7c5d2f7`).

The "no mocks" rule (CLAUDE.md §3, project-level) covers spirit, not
just imports — `FakeArgs` is a hand-rolled stub for argparse output.
The acceptable replacement is a real `argparse.Namespace(...)`
instance, e.g.:

```python
import argparse
args = argparse.Namespace(path='content/hero.md',
                          directory=self.directory,
                          token=None,
                          base_url=None,
                          id=True,
                          json=False)
vault.cmd_cat(args)
```

This is a real object (literally what `argparse.parse_args()` returns)
and removes the stub. It also removes the four duplicated `class
FakeArgs` blocks.

## Severity rationale

- **major (not blocker)** — no new sprint code introduced
  `unittest.mock` or `MagicMock` imports. The carryover violations
  (553 mock lines, 519 monkeypatch lines) substantially exceed the
  v0.5.11 baseline of 18 violations, but this sprint did not worsen
  that count except for the 4 `FakeArgs` stubs.
- The `FakeArgs` regression is small but symbolic — it shows the
  no-mocks discipline can slip under deadline pressure. Treat as a
  Phase 3 cleanup target.

## Recommended fixes (Villager Dev scope)

1. Replace 4 × `class FakeArgs` in `test_CLI__Vault__Write.py` with
   `argparse.Namespace(...)` calls.
2. Defer the 553-line carryover backlog to the existing Phase 3
   refactor (do not touch in this analysis pass).

## Escalations

- **Sherpa / Architect** — confirm baseline-delta target. Should the
  v0.5.11 baseline be re-counted (553 vs. 18 looks like double-counting
  or a definition change between sprints)? If 18 was unique-occurrences
  and 553 is line-matches, the comparable figure should be lower —
  but even uniques are likely > 100. Recommend a one-sprint mock-
  removal initiative tracked by Architect.
- **AppSec** — `MagicMock` returns from `patch.object(...,
  load_branch_index, return_value=MagicMock())` pass attribute
  assertions silently. Any test that "validates" key-handling using
  this pattern is a security-test smell. Listed for AppSec review.
