## Answer: Missing Dependencies

**Assume `sgraph_ai_app_send` is available at runtime.** Do not write graceful skips — the dependency will be installed. The QA tests must run against a real, local SG/Send server.

---

## How It Works

The `sgraph-ai-app-send` package provides `setup__send_user_lambda__test_server()` which spins up the **full SG/Send User Lambda** as a real HTTP server on a random port, using in-memory storage. This is the same FastAPI app that runs in production on AWS Lambda — the only difference is `Storage_FS__Memory` instead of S3.

- Starts in ~100ms
- Real TCP on `127.0.0.1:{random_port}`
- All vault endpoints: write, read, batch, list, delete, `write-if-match` (CAS)
- In-memory storage — no S3, no AWS credentials, no external dependencies
- Context manager — clean slate per test session, auto-cleanup on exit

---

## The Fixture Pattern

Your `tests/qa/v030/conftest.py` should use this pattern:

```python
import pytest

from sgraph_ai_app_send.lambda__user.testing.Send__User_Lambda__Test_Server import setup__send_user_lambda__test_server

@pytest.fixture(scope='session')
def send_server():
    """Start a real SG/Send User Lambda on a random port (in-memory storage)."""
    with setup__send_user_lambda__test_server() as test_objs:
        yield test_objs
```

This gives you:

| Field | Value | Use For |
|-------|-------|---------|
| `send_server.server_url` | `http://127.0.0.1:{port}` | Base URL for Playwright to hit |
| `send_server.access_token` | Random GUID | `x-sgraph-access-token` header |
| `send_server.write_key` | Random GUID | `x-sgraph-vault-write-key` header |

---

## Authentication Headers

| Header | Value | Required For |
|--------|-------|-------------|
| `x-sgraph-access-token` | `send_server.access_token` | Write, delete, batch |
| `x-sgraph-vault-write-key` | `send_server.write_key` | Write, delete, batch |

Read and list endpoints require **no authentication** (zero-knowledge model).

## Python 3.12 Requirement

`sgraph-ai-app-send` requires Python >= 3.12. The QA tests must run in the 3.12 venv:

```bash
# Setup (one-time)
python3.12 -m venv /tmp/sgit-ai-venv-312
/tmp/sgit-ai-venv-312/bin/pip install -e ".[dev]"
/tmp/sgit-ai-venv-312/bin/pip install sgraph-ai-app-send

# Run QA tests
/tmp/sgit-ai-venv-312/bin/python -m pytest tests/qa/v030/ -v
```

---

## Key Point for Playwright Tests

The `send_server.server_url` is what your UI should be configured to talk to. Point the frontend at this URL instead of the production endpoint. The server is real HTTP on localhost — Playwright can interact with the UI normally while the UI makes real API calls to this local server.
