## E. Suspected duplicate-setup hotspots

Sampled the slowest files (read-only) to confirm what's expensive.

1. **`tests/unit/cli/test_CLI__PKI.py`** (5.25s setup, 28 tests, ~0.19s/test setup average; some setups hit 1.07s)
   Spot-read shows every test class uses `setup_method` to call `tempfile.mkdtemp()` + `CLI__PKI().setup(sg_send_dir=...)`. Some tests then call `cmd_keygen` which generates a fresh **RSA-OAEP 4096-bit key pair + ECDSA P-256** keypair, and PBKDF2-encrypts the private key. RSA-4096 keygen alone is ~0.5–1.0s on commodity CPUs. Many tests do not need a unique key pair.
   **Suspected expensive op:** RSA-4096 keypair generation + PBKDF2 wrapping, repeated per-test.

2. **`tests/unit/sync/test_Vault__Sync__Probe.py`** (4.48s setup, 9 tests; two setups at 2.28s and 2.20s)
   Spot-read confirms `Test_Vault__Sync__Probe` and `Test_Vault__Sync__Probe__JSON` already use `setup_class` + `Vault__Test_Env().setup_single_vault(...)` with a single first-push that uploads bare structure. The 2.2s cost is the class-scoped vault build (key derivation from a simple token + vault init + first commit/push), borne once per class but appearing on the first test of the class. Class-level sharing already exists; further speedup would need session-level sharing across the two probe classes.
   **Suspected expensive op:** simple-token key derivation + first-push upload of branch index in `Vault__Test_Env`.

3. **`tests/unit/sync/test_Vault__Bare.py`** (3.65s setup over 18 tests = ~0.20s/test, call ≈ 0)
   Spot-read shows `setup_method` does `tempfile.mkdtemp()` + `Vault__Sync.init` + writes 2 files + `Vault__Sync.commit('add test files')` + key derivation + ref-write — **for every single test**, even though most assertions just check `is_bare()` or `checkout` against the same constructed bare vault. The work is identical across tests; only the assertion differs.
   **Suspected expensive op:** per-test full vault construction (init + commit + key derivation + ref write) where a class- or session-scoped fixture would be safe.

4. **`tests/unit/sync/test_Vault__Sync__Simple_Token.py`** (call-dominated 11.70s, top-2 individual tests)
   Spot-read shows `setup_method` recreates `Vault__Crypto`, `Vault__API__In_Memory`, `API__Transfer__In_Memory` per test. The expensive bits are inside `call`: `clone_simple_token_vault_found` and `_clone_has_simple_token_config` each end-to-end-clone a vault, which derives keys from a simple token (intentional CPU cost) and round-trips through the in-memory transfer API. These are real round-trips — not setup duplication, but candidates for shared "pre-cloned vault" fixtures if the assertion only inspects post-clone state.
   **Suspected expensive op:** per-test simple-token key derivation + clone round-trip.

5. **`tests/unit/transfer/test_Vault__Archive.py`** (call-dominated 10.51s over 28 tests)
   Most cost is in `call` — the archive build/round-trip is the unit under test, so this is harder to factor out. Some helper-like tests (e.g. `test_file_count_in_manifest` at 0.74s) build a full archive only to read one number out of the manifest; cheaper shared-archive fixtures would help.
   **Suspected expensive op:** `build_inner_zip` + `Vault__Crypto.encrypt` over `MULTI_FILES`.

**In-memory transfer server boot:** sampled files use lightweight `Vault__API__In_Memory` / `API__Transfer__In_Memory` (dicts), not a real HTTP server boot. Cost is negligible.

**Real temp dirs:** every slow file uses `tempfile.mkdtemp()` per `setup_method` — the directory creation itself is cheap (<1ms), but the work that fills it (vault init, key write, file write) is what costs.

**Top-3 shared-fixture candidates for brief 03:**

1. **PKI keypair fixture** for `test_CLI__PKI.py` (and any other PKI consumer) — generate one RSA-4096+ECDSA-P256 keypair + wrapped private key once per session; tests that don't mutate it share. Saves ~5s.
2. **Constructed bare vault** for `test_Vault__Bare.py` — class-scoped vault with 2 files committed + ref written; restore-from-snapshot per method. Saves ~3.5s.
3. **Pre-cloned simple-token vault** for `test_Vault__Sync__Simple_Token.py` and `test_Vault__Sync__Probe.py` — one already-cloned vault state shared across "inspection-only" tests; tests that mutate clone keep their own. Saves ~5–6s. (Probe already does class-level sharing; extending to session would cut another ~2s.)
