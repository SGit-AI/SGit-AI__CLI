## E. Specifically-flagged-by-deep-analysis paths

### 1. `sgit_ai/crypto/Vault__Crypto.py` — `encrypt_deterministic` family

**Status: COVERED.** Vault__Crypto.py is at 99.2%; only line 50 is missing.
Lines 162-176 (`encrypt_deterministic` + `encrypt_metadata_deterministic`)
are fully exercised. The cross-team note that these had "zero direct tests"
no longer holds for v0.10.30 — they are reached transitively via the tree
object encryption path. **Finding: known gap is closed at the line-coverage
level.** A dedicated round-trip / determinism test vector is still missing
(no test asserts that two calls with the same key+plaintext produce the same
ciphertext directly), so brief 18 (`crypto-determinism-tests`) remains
worthwhile, but this is a *behavioural* gap, not a coverage gap.

### 2. `sgit_ai/sync/Vault__Sync.py` — checkpoint triplet + probe_token

- `_load_push_state` (lines 2729-2740): **2732-2739 missing.** Confirmed
  uncovered. Only the no-file return path (line 2740) is reached.
- `_save_push_state` (lines 2742-2744): **2743-2744 missing.** Confirmed
  uncovered.
- `_clear_push_state` (lines 2746-2748): **2747-2748 missing.** Confirmed
  uncovered.
- `probe_token` share-token branch (lines 1821-1830): **1821, 1822, 1830
  missing.** Confirmed: only the vault branch is taken in tests; the
  fallback to `API__Transfer.info()` for share-token detection is dead
  code in the test suite.

### 3. `bare/data/{blob_id}` ciphertext-vs-plaintext assertion

**Confirmed missing.** `tests/unit/sync/test_Vault__Sync__Write_File.py`
makes no assertion on the bytes stored at `bare/data/<blob_id>` after
`write_file` runs — no `ciphertext` or `plaintext` token appears in the
file. Other tests touch `bare/data/...` paths but only as opaque
identifiers. The "no plaintext on disk after write_file" property is
not covered by any unit test.

## F. Delta vs v0.5.11

| Metric         | v0.5.11 | v0.10.30 |  Delta |
|----------------|--------:|---------:|-------:|
| Tests passed   |     430 |    2,105 |  +1,675 |
| Tests skipped  |      28 |        0 |    -28 |
| Total stmts    |   2,381 |    9,908 |  +7,527 |
| Stmts covered  |   1,981 |    8,484 |  +6,503 |
| Coverage %     |     83% |      86% |    +3pp |
| Suite runtime  |    ~53s |  ~134s   |   +81s |

Notes:

- v0.5.11 used package `sg_send_cli`; v0.10.30 uses `sgit_ai`. Comparison is
  approximate — the rename also pulled in transfer/objects/cli refactors.
- Statement count more than quadrupled (+316%), test count nearly quintupled
  (+390%), yet line coverage rose only 3 pp. Net new code is being landed at
  roughly the previous coverage rate, not above it.
- All 28 v0.5.11 skipped tests are gone — either deleted, ported, or moved
  to integration (`tests/integration/`, not run here per CLAUDE.md).
- `Vault__Sync.py` grew from 411 statements (90%) to **1,948 statements
  (74.7%)** — single biggest absolute uncovered block in the codebase
  (492 missing lines). This file alone accounts for ~35% of the entire
  uncovered surface.
- `cli/CLI__Vault.py` grew from 274 stmts (66%) to **1,111 stmts (66%)** —
  same ratio, 4× the absolute miss (382 lines). Largest single uncovered
  block in `cli/`.
- The two v0.5.11 files at 0% (`Schema__PKI_Key_Pair`, `Schema__PKI_Public_Key`)
  are now both at 100%.
- New 0%-coverage file appeared: `sgit_ai/api/Vault__Backend__API.py`
  (21 stmts).
