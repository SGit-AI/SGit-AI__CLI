## 3. Visual Direction

### Token Usage

All colours, typography, spacing, and radii come from `sg-tokens.css`. No custom values.

| Token category | Primary tokens used |
|---------------|-------------------|
| **Surface** | `--sg-color-surface-default`, `--sg-color-surface-dark`, `--sg-color-surface-elevated` |
| **Text** | `--sg-color-text-primary`, `--sg-color-text-secondary`, `--sg-color-text-inverse`, `--sg-color-text-muted` |
| **Accent** | `--sg-color-accent-green` (CTAs, terminal prompts, success), `--sg-color-accent-blue` (links, icons) |
| **Typography** | `--sg-font-display` (headlines), `--sg-font-body` (prose), `--sg-font-mono` (code/terminal) |
| **Spacing** | `--sg-space-xs` through `--sg-space-3xl` for all margins/padding |
| **Radius** | `--sg-radius-sm` (buttons, inline code), `--sg-radius-md` (cards), `--sg-radius-lg` (terminal blocks) |

### CLI / Terminal Aesthetic

The terminal is the hero visual element across the site. Design guidelines:

1. **Terminal blocks** use `--sg-color-surface-dark` background, `--sg-radius-lg`, with a top bar showing three dot "traffic lights" (decorative, CSS-only) and a title bar
2. **Prompt character** is `$` in `--sg-color-accent-green`
3. **Command text** is `--sg-color-text-inverse` bold
4. **Output text** is `--sg-color-text-muted-inverse` regular weight
5. **All code blocks site-wide** use this terminal style, not generic `<pre>` styling

### Security Theme

- The privacy badge ("All encryption happens on your device") appears in the footer of every page and in the hero section
- Lock icon (SVG) accompanies any mention of encryption operations
- The colour palette is intentionally muted and professional — no bright marketing colours. Trust is communicated through restraint
- Technical accuracy in all diagrams. Crypto parameters are stated precisely (AES-256-GCM, not "military-grade encryption")

## 5. Interactive Demo Concepts

### 5a. Animated Terminal Demo (`<sg-terminal-demo>`)

**Location:** Landing page hero (Section A) and Demos page.

**Behaviour:**
1. Component accepts a JSON script array: `[{ "type": "command", "text": "sgit init my-vault", "delay": 50 }, { "type": "output", "text": "Initialized encrypted vault...", "delay": 10 }, ...]`
2. Commands are typed character-by-character (configurable speed via `delay` in ms per character)
3. Output lines appear instantly after a brief pause (300ms)
4. Cursor blinks at prompt between commands (standard CSS `@keyframes blink`)
5. Auto-plays on viewport intersection (`IntersectionObserver`). Pauses when out of view
6. Manual controls: play/pause button, restart button (bottom-right of terminal frame)
7. Loops after completion with a 3-second pause

**Demo script for landing page (5 commands):**

```
$ sgit init my-vault
Initialized empty vault in my-vault/
  Vault ID:  a1b2c3d4
  Vault key: correct-horse:a1b2c3d4

$ cp secrets.json my-vault/
$ sgit commit "add secrets" -d my-vault
[clone-xyz] add secrets
  commit 8f3e1a2b

$ sgit push my-vault
Pushing to remote... done
Pushed 1 commit(s), 2 object(s) uploaded.

$ sgit clone correct-horse:a1b2c3d4
Cloning into 'a1b2c3d4'...
Cloned into a1b2c3d4/
  Vault ID:  a1b2c3d4
  HEAD:      8f3e1a2b
```

**Sizing:** 600px wide x 360px tall on desktop. Full-width on mobile with reduced font size.

### 5b. Crypto Flow Diagram (`<sg-crypto-flow>`)

**Location:** Demos page, Features page (encryption section), Docs crypto page.

**Visual:** Horizontal pipeline diagram (SVG), animated left-to-right:

```
Vault Key ──► PBKDF2 ──► Master Key ──► HKDF ──► Per-File Key ──► AES-256-GCM ──► Ciphertext
  (user)      (600k)      (256-bit)     (SHA256)   (unique/file)    (encrypt)       (to server)
```

**Animation:**
1. Data flows left-to-right as coloured dots/pulses along the pipeline
2. Each stage highlights on hover, showing a tooltip with parameters (e.g., "PBKDF2-SHA256, 600,000 iterations, 32-byte output")
3. The final "to server" node has a lock icon and the text "server sees only ciphertext"
4. Animate on scroll intersection. Static fallback for `prefers-reduced-motion`

**Implementation:** Inline SVG with CSS animations. No canvas, no external libraries.

### 5c. Demos Page Additional Concepts

| Demo | Type | Description |
|------|------|-------------|
| Vault workflow | `<sg-terminal-demo>` | Extended 10-step script: init, add files, commit, push, clone on second machine, modify, commit, push, pull on first machine, log --graph |

## 6. Responsive Design Approach

### Breakpoints

Use `sg-tokens.css` breakpoints. Fallback values if tokens do not define them:

| Name | Min-width | Usage |
|------|-----------|-------|
| `sm` | 640px | Single column to 2-column transition |
| `md` | 768px | Docs sidebar appears |
| `lg` | 1024px | 3-column feature grid, hero side-by-side |
| `xl` | 1280px | Max content width reached, centred |

### Mobile-First Rules

1. **Base styles are mobile.** All `min-width` media queries add complexity upward. No `max-width` queries.
2. **Hero:** Stacked (headline above terminal demo). Terminal demo is full-width, reduced to `--sg-font-size-sm` monospace.
3. **Feature cards:** Single column stack. Cards get full width with `--sg-space-md` vertical gap.
4. **Ecosystem cards:** Horizontal scroll on mobile (CSS `overflow-x: auto`, `scroll-snap-type: x mandatory`). No wrapping.
5. **Terminal blocks:** Full-width with horizontal scroll for long lines. No line wrapping on commands.
6. **Docs sidebar:** Hidden below `md`. Toggled via hamburger button that slides sidebar in as overlay.
7. **Crypto flow diagram:** Stacks vertically on mobile (pipeline becomes top-to-bottom). SVG `viewBox` adapts.
8. **Touch targets:** All interactive elements have minimum 44x44px touch area.
9. **Font scaling:** Body text is `--sg-font-size-base` (16px). No text smaller than 14px on mobile.

### Performance

- No images above the fold except inline SVG icons. Terminal demo is pure DOM + CSS.
- `sg-tokens.css` loaded in `<head>`. Component JS loaded with `type="module"` (deferred by default).
- Total landing page target: <50KB transferred (HTML + CSS + JS, excluding shared header/footer).
