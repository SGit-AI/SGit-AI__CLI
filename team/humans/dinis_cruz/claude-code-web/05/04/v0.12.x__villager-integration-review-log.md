Five-hypothesis verdict against a 43-commit / 330-tree synthetic vault:

| Hypothesis | Verdict | Key number |
|---|---|---|
| H1 — many small BFS waves | MINOR | 3–4 waves, avg batch ~600 objects — waves are large |
| H2 — decrypt/parse overhead | MINOR | 0.79 ms/tree → ~1.9s for 2,375 trees (1% of 184s) |
| H3 — random-IV no-dedup | **PRIMARY CAUSE** | Synthetic 5.35× dedup; real vault ~1.0× (all objects unique) |
| H4 — server-side S3 GETs per object | LIKELY SIGNIFICANT | ~120s estimated; unverifiable without backend |
| H5 — walking history unnecessary | **CONFIRMED lever** | HEAD-only = 12.7% of trees; 87% waste |

Root cause: old vaults used random-IV AES-GCM → every tree re-encrypted per commit →
`visited_trees` set cannot dedup → 2,375 downloads instead of ~300.

Recommended fix priority: H5+B08 (HEAD-only pack, ~40–100× speedup) → H3 migration (B10, ~7–8×) → H4 (B08 batching).

### Fixes applied (shared fix commit `ced6038` covers B02–B05)

**`Workflow__Workspace._atomic_write` — `@staticmethod` violation:**
CLAUDE.md bans `@staticmethod`. Removed decorator; method signature gains `self` (callers already used `self._atomic_write(…)` so no call-site changes needed).

**`Workflow__Runner._now_iso` / `_now_ms` — module-level functions:**
Two helper functions at the bottom of `Workflow__Runner.py` violate "no module-level functions". Moved into the class as private instance methods; 6 call sites updated to `self._now_iso()` / `self._now_ms()`.

**`CLI__Dev__Workflow` — 3 module-level functions + 1 module-level `dict`:**
`register_workflow`, `_known_workflows`, `_find_workspace`, and `_WORKFLOW_REGISTRY` were all at module level. Moved into `CLI__Dev__Workflow` as `@classmethod` / instance method + class-level `_registry = {}` attribute. Module-level aliases (`register_workflow = CLI__Dev__Workflow.register_workflow`, `_WORKFLOW_REGISTRY = CLI__Dev__Workflow._registry`) kept so existing test imports (`from … import register_workflow, _WORKFLOW_REGISTRY`) continue to work without changes.

**`test_Step.py`, `test_Workflow__Workspace.py`, `test_Synthetic__Workflow.py` — raw `str` fields in Type_Safe classes:**
7 test-fixture Type_Safe classes had `field : str = None` — Type_Safe violation. Changed to `field : Safe_Str = None`. `Safe_Str` sanitises hyphens → underscores, so test values containing hyphens (`'result-a'`, `'pre-done'`, `'step2-out'`) were updated to underscore equivalents.
