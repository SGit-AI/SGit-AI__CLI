## What Was Built

### Phase A — Measurement (Brief 01 + 02)

starting point and identified the worst-covered files:
- `Vault__Backend__API.py` — 0%
- `Vault__Sync__Clone.py`, `Vault__Sync__Pull.py`, `Vault__Sync__Push.py` — 50-70%
- `Vault__Sync__Status.py` — low coverage

### Phase B — Test Infrastructure (Briefs 03–06 + test-framework pack)

**Shared fixtures (`tests/conftest.py`, `tests/_helpers/vault_test_env.py`):**
Six session/module-scoped snapshots eliminate per-test vault setup overhead and
make test state explicit and reproducible:
- `known_test_keys` — pre-derived keys for 5 canonical vault keys
- `precomputed_encrypted_blobs` — pre-encrypted blobs (small/medium/large)
- `two_clones_pushed` — Alice init+push, Bob clone snapshot
- `vault_with_N_commits` — parameterised N={1,5,20} commit history
- `vault_with_pending_changes_snapshot` — dirty working copy
- `vault_with_branches_snapshot` — two-branch diverged vault
- `read_only_clone_snapshot` — read-only clone via `clone_read_only`

`Vault__Test_Env` class provides `setup_*` + `restore()` pattern for
class-level snapshot isolation without module-scoped fixtures.

**CI parallelisation:** `pytest-xdist` two-pass shape wired in (`-n auto`
parallel pass + serial `no_parallel` pass). ≤5 tests marked `no_parallel`.

### Structural Refactor — Brief 22 (Vault__Sync split)

`Vault__Sync.py` reduced from **3,032 LOC → 161 LOC** (facade only).
Ten sub-classes extracted:

`Vault__Sync__Base` · `Vault__Sync__Commit` · `Vault__Sync__Pull` ·
`Vault__Sync__Push` · `Vault__Sync__Clone` · `Vault__Sync__Status` ·
`Vault__Sync__Branch_Ops` · `Vault__Sync__Lifecycle` · `Vault__Sync__GC_Ops` ·
`Vault__Sync__Sparse` · `Vault__Sync__Fsck` · `Vault__Sync__Admin`

All public methods now delegate: `Vault__Sync__XYZ(crypto=self.crypto,
api=self.api).method(...)`. Each sub-class is independently testable.

### Bug Fixes

- **`delete_on_remote` stale push_state** (Brief 14, Architect finding 06):
  `_clear_push_state` not called after remote deletion — next push to the same
  vault ID would skip Phase-A blob upload, producing dangling references.
  Fixed; tests document failure mode and verify the fix end-to-end.

- **`reset()` NameError** (`Vault__Sync__Pull.py`): `vault_commit` was
  referenced before definition (pre-existing `# noqa: F821`). Fixed by
  constructing `Vault__Commit` before first use.

- **`destroy` missing access token** (`Vault__API.py`): `x-sgraph-access-token`
  header was absent from the DELETE request. Added.

- **Dead code removed** (`Vault__Sync__Status.py`): Two mathematically
  unreachable `else` branches proven dead via BFS-walk analysis and removed,
  reducing complexity and bringing Status to 100% coverage.
