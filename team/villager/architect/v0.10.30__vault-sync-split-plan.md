## 3. Roll-back protocol

- Each extraction is exactly one `git commit`. Revert = `git revert <sha>`.
- No extraction commit may touch more than two files (source + test).
- If any extraction causes suite failure: revert immediately, do not attempt
  to fix forward in the same commit.
- E5 is structured as one commit per sub-module. Each sub-module's commit
  must leave the full suite green before the next sub-module starts.

---

## 4. Acceptance per extraction

All gates apply at **every commit boundary**:

1. `pytest tests/unit/ -q` → 2,333+ tests pass (0 failures).
2. `pytest --cov=sgit_ai` → coverage ≥ 88% (current baseline).
3. No CLI command output changes (validated by existing CLI test suite).
4. `sgit_ai/sync/Vault__Sync.py` shrinks by at least the "LOC moved" figure.

After E5: `Vault__Sync.py` must be ≤ 1,000 LOC; no new file may exceed
1,000 LOC.
