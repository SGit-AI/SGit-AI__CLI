### 1c. `tests/unit/sync/test_Vault__Sync__Probe.py` — 8.63s / 9 tests

### 1d. `tests/unit/sync/test_Vault__Sync__Simple_Token.py` — 11.70s / 10 tests, call-dominated

Cost is in `call`, not `setup`. Top-2 tests
(`test_clone_simple_token_vault_found`, `…_clone_has_simple_token_config`)
each do an end-to-end init+commit+push+clone. Both assert on the same
post-clone state. Other 8 tests exercise `init`, error paths, or
`clone_from_transfer` — each genuinely needs fresh state.
**Share-safety: 2/10 (20%)** — below the 70% threshold for a class-wide
fixture, but those 2 tests are the entire 6.57s hot path. Carve a narrow
module-scoped fixture for just those two; leave the rest alone.

## Section 2 — Fixture catalogue

Five fixtures, all real objects, no mocks. Total estimated saving **~12–13s**
(124s → ~111s).

### F1. `pki_keypair_snapshot` — module scope, `tests/unit/cli/conftest.py`
Builds one RSA-4096 + ECDSA-P256 keypair once per module via
`CLI__PKI().setup() + key_store.generate_and_store`. Returns `{snapshot_dir,
metadata}`. **Mutation contract:** snapshot is read-only; never mutated by
tests. Build cost ~1.0s. Serves Export_Delete (4) + Import_Contacts (5).
Saving ~3.5s.

### F2. `pki_workdir` — function scope factory, same conftest
Callable: `def make() -> (tmp_dir, CLI__PKI)`. `copytree`s the snapshot's
`keys/` and `keyring/` into a fresh tempdir; returns a wired `CLI__PKI`.
Build cost ~3 ms. Mutation contract: workdir is the test's own.

### F3. `bare_vault_snapshot` — module scope, `tests/unit/sync/conftest.py`
Two named variants under one fixture (different file sets per class):
`small_vault` (config.json + deploy/run.sh) and `read_list_vault`
(readme.txt + docs/guide.md). Each snapshot is a golden directory + the
`vault_key` string. Build cost ~0.4s × 2 once. Serves all 18 tests in
`test_Vault__Bare.py`. Saving ~3.4s.

### F4. `bare_vault_workspace` — function scope factory, same conftest
`def make(name) -> (tmp_dir, vault_key, crypto, bare, sync)`. `copytree`s
the named F3 snapshot. ~3 ms.

### F5. `probe_vault_env` — session scope, same conftest
Wraps existing `Vault__Test_Env.setup_single_vault('give-foul-8361',
{'readme.md': 'probe test vault'})`. Tests call `.restore()` per-test
(existing API). Serves both Probe classes; their `setup_class` blocks are
deleted. Build cost ~2.2s once. Saving ~2.2s.

### F6. `simple_token_origin_pushed` — module scope, same conftest
Snapshots a post-init+commit+push origin for `TOKEN_SIMPLE`. Consumed by
the **only two** share-safe tests in `test_Vault__Sync__Simple_Token.py`;
each clones into its own fresh tempdir. Saving ~2.5–3s.

(Optional F1b: promote F1 to session scope and replace the existing
`setup_class` blocks in PKI Sign_Verify / Encrypt_Decrypt. Adds ~1s saving
but increases review surface. **Defer; brief 04 may skip.**)

## Section 3 — Isolation contract

**Mechanism: snapshot + per-test `shutil.copytree` clone** (plus
`copy.deepcopy` of in-memory `Vault__API__In_Memory._store` for F5).
Already proven in `Vault__Test_Env` and PKI Sign_Verify/Encrypt_Decrypt.

Justification:
- Every fixture's expensive object is a directory tree on disk plus an
  in-memory dict. Both clone cleanly.
- Snapshot dirs live inside fixture-private tempdirs; **no test ever opens
  them for write**. A future agent who tries to rely on cross-test mutation
  will mutate their own clone, not the snapshot — the assertion will fail
  in the next test.
- Type_Safe's `freeze()` does not extend to filesystem state, so
  snapshot+clone is the only mechanism that actually works for vault
  directories.
- Factory-only rebuild (cached intermediates) defeats the optimisation —
  PBKDF2 + RSA-4096 + commit hashing is exactly what we're avoiding.

currently in scope) MUST stay un-shared.

## Section 5 — Conftest layout

```
tests/
  conftest.py                  (existing — unchanged)
  unit/
    cli/conftest.py            (NEW — F1, F2, optional F1b)
    sync/conftest.py           (NEW — F3, F4, F5, F6)
    sync/vault_test_env.py     (existing — unchanged; F5 wraps it)
```

- **No `tests/unit/conftest.py`** — fixtures stay subdir-scoped.
- **No `__init__.py` anywhere under `tests/`** (project rule). Confirmed
  empty; existing 10 files importing `from tests.unit.sync.vault_test_env
  import …` already work via pytest's rootdir auto-add.
- **Naming:** prefixes (`pki_*`, `bare_vault_*`, `probe_vault_*`,
  `simple_token_*`) keep grep-ability and avoid cross-conftest collision
  if scope ever broadens.

## Section 6 — Risks and verification

### Risks
1. **Stale snapshot** — SUT change reflects old behaviour. *Mitigation:*
   built every session via real `Vault__Sync.init/commit/push`; no pickling.
2. **Hidden mutation of `snapshot_dir`.** *Detection:* reverse-order run
   (verification step 5). `pytest-randomly` is NOT a dev dep; do not add.
3. **xdist race in brief 05.** All fixtures are build-once-immutable or
   factory; no post-build mutation. Safe for xdist.
4. **F3 has two variants.** Don't promote to session without re-checking
   that consumers explicitly select a variant.
5. **F5 layered on `Vault__Test_Env`.** Other 9 consumers keep their own
   instances; only Probe migrates.

### Verification commands for brief 04's executor

```bash
# 1. Pass count must be unchanged
python -m pytest tests/unit/ -q 2>&1 | tail -2   # expect "2105 passed"

# 2. Coverage must not regress
python -m pytest tests/unit/ --cov=sgit_ai --cov-report=term 2>&1 | tail -2
#   expect TOTAL ... 86%

# 3. Wall clock improves
time python -m pytest tests/unit/ -q              # expect ≤ 115s

# 4. Per-file isolation (snapshot pollution detector)
for f in tests/unit/cli/test_CLI__PKI.py \
         tests/unit/sync/test_Vault__Bare.py \
         tests/unit/sync/test_Vault__Sync__Probe.py \
         tests/unit/sync/test_Vault__Sync__Simple_Token.py; do
  python -m pytest "$f" -q || echo "FAIL: $f"
done

# 5. Reverse-order run (no new dependency required)
python -m pytest tests/unit/cli/test_CLI__PKI.py --collect-only -q \
  | tac | head -30  # eyeball; manually --deselect/--select if drift suspected

# 6. Read-only constraint after refactor
git diff -- sgit_ai/ pyproject.toml pytest.ini .coveragerc   # must be empty
```
