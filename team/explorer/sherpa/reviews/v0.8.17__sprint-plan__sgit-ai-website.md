## 1. MVP Scope

### v1 — What ships first

The MVP is a static website at sgit.ai that establishes SGit's identity within the SGraph-AI ecosystem, communicates the value proposition to developers, and provides a clear path to installation and first use.

**In scope for v1:**

| Page / Section | Purpose |
|----------------|---------|
| **Landing page** | Hero with value proposition, install command (`pip install sgit-ai`), quick-start flow |
| **Feature overview** | Four pillars: vault sync (clone/commit/push/pull), encryption (AES-256-GCM, PBKDF2, HKDF), PKI (RSA-4096 + ECDSA P-256), inspection (log, fsck, cat-object) |
| **Ecosystem section** | "Part of the SGraph-AI family" — links to send.sgraph.ai, vault.sgraph.ai, tools.sgraph.ai |
| **SG/Vault case study** | "SG/Vault powered by sgit" — the primary proof point showing sgit in production use |
| **Getting started** | Install, init, add files, commit, push — a 5-step quick-start on the landing page |
| **Shared header/footer** | sg-site-header, sg-site-footer from SGraph-AI__Tools |
| **Cross-site navigation** | sg-nav.js integration for ecosystem-wide navigation |
| **Responsive layout** | Mobile-first, works on all screen sizes |

**Technical constraints for v1:**
- Static HTML/CSS/JS only — no server-side dependencies
- Vanilla JS, ES modules, no build step
- Shared SGraph-AI design tokens (sg-tokens.css)
- S3/CloudFront deployment (same pattern as tools.sgraph.ai)

### v2 — What waits

| Feature | Reason to defer |
|---------|----------------|
| Interactive terminal demo (Pyodide-based sgit in browser) | High effort; needs sg-send-cli pattern adapted |
| Full documentation site (all CLI commands with examples) | Content volume; landing page quick-start is sufficient for MVP |
| Blog / changelog page | No content yet; version history can link to GitHub releases |
| i18n (17-locale system) | Content must stabilize before translation |
| Community resources page | Community too early-stage |
| Crypto visualization (key derivation flow diagram) | Valuable but not launch-critical |
| Search functionality | Requires documentation volume to justify |

## 4. Critical Path

The critical path runs through the content pipeline:

```
1.1 (value prop) → 1.6 (site map) → 2.1 (landing layout) → 3.1 (scaffold) → 3.2 (landing page) → 3.7 (a11y) → 4.6 (staging) → 5.6 (launch)
```

**Bottleneck analysis:**

| Bottleneck | Risk | Mitigation |
|------------|------|------------|
| Ambassador copy delivery (M1) | Blocks all downstream work | Ambassador starts immediately; Designer can sketch wireframes before copy is final |
| Designer specs (M2) | Blocks Dev implementation | Use SGraph-AI ecosystem patterns to reduce design decisions; leverage existing sg-tokens.css |
| SGraph-AI shared component availability | External dependency | Verify component CDN URLs and versions exist before M3 starts |
| DNS/TLS provisioning (M4) | Can delay launch | Start DevOps infrastructure work during M2 — no design dependency |

**Schedule compression opportunities:**
- DevOps can provision infrastructure (4.1-4.4) in parallel with M1-M3
- Designer can create wireframe sketches during M1 before final copy
- Dev can set up project scaffold (3.1) as soon as component inventory (2.3) is ready, before all design specs complete

### SG/Vault Case Study — Integration Detail

The SG/Vault case study is the primary proof point for sgit-ai. It demonstrates:

1. **Vault creation** — `sgit init` creates the encrypted vault structure
2. **Content sync** — `sgit commit` + `sgit push` sends encrypted objects to SG/Send Transfer API
3. **Multi-device access** — `sgit clone` retrieves and decrypts on another device
4. **Branch model** — Named branches for collaboration, clone branches for local work
5. **Zero-knowledge guarantee** — Server never sees plaintext; all crypto is client-side (Web Crypto interop)
6. **Browser interop** — Same vault can be opened in vault.sgraph.ai (browser) or sgit CLI — AES-256-GCM output is byte-for-byte identical

This case study connects sgit.ai to the broader ecosystem and should be prominently featured on the landing page.
