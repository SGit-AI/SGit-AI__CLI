## Brief 12 — clear_kdf_cache for PBKDF2 LRU residency (2026-05-01)

**AppSec findings closed:** F03 (PBKDF2 LRU passphrase residency — MEDIUM CLI / HIGH agent).

Added `clear_kdf_cache()` method to `Vault__Crypto` (`sgit_ai/crypto/Vault__Crypto.py:185–199`).  The method calls `_pbkdf2_cached.cache_clear()` on the module-level `functools.lru_cache(maxsize=256)` that caches PBKDF2-SHA256 derivations.  For one-shot CLI runs the cache is harmless; for long-running agent processes the cache would retain passphrase bytes and derived key material indefinitely.  Clearing at natural passphrase-boundary points closes the residency window.

Wired into three locations in `Vault__Sync`:

| Method | When cleared |
|---|---|
| `delete_on_remote` | After `api.delete_vault()` returns — vault is gone, key is stale |
| `rekey_wipe` | After `secure_rmtree` — old vault material is wiped, old key is stale |
| `probe_token` | After each return path (success vault, success share, failure) — probe must leave zero key material in process state |

New test file: `tests/unit/crypto/test_Vault__Crypto__KDF_Cache.py` — 3 tests: `test_clear_kdf_cache__functional` (clear empties cache, subsequent derivations still produce correct key), `test_clear_kdf_cache__currsize_zero_after_clear` (cache_info().currsize == 0 immediately after clear), `test_pbkdf2_cache_size_bounded` (M5 mutation closer — asserts maxsize==256 and currsize grows, both fail if maxsize=0).

Mutation matrix M5 row updated from "U" to "D".  Suite result: all 3 new tests pass.  No behaviour change — the KDF itself is untouched.
