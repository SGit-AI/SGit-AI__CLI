## Standing review checklist (applied on every merge)

- [ ] No multi-paragraph **class or method** docstrings — one line max (module-level docstrings are fine)
- [ ] No bare `ClassName()` instantiations just to call a utility method
- [ ] No duplicated helper methods across layers
- [ ] No mocks / patches / monkeypatch (`unittest.mock`, `MagicMock`, `@patch`) — use real objects
- [ ] No `__init__.py` files under `tests/`
- [ ] No Type_Safe violations: raw `str`/`int`/`dict` fields in data classes (`bool` is allowed)
- [ ] No module-level functions (`def foo(` at module scope outside a class)
- [ ] No `@staticmethod` decorators
- [ ] No new layer-import violations beyond the 7 KNOWN_VIOLATIONS
- [ ] No upward import from `sgit_ai/` to `sgit_show/` (v0.13.x rule)
- [ ] No backend changes — everything must be client-side (v0.13.x rule)
- [ ] No commits touching archived briefs (`v0.12.x__perf-brief-pack/archived/`)

### Notes on approach

- `_FakeApiNoBranches` encrypts a real empty `Schema__Branch_Index` using the test's `read_key_bytes` so the branch manager can decrypt it and return 0 branches naturally.
- `_FakeApiNoRefs` delegates index calls to the real API but filters out `bare/refs/` and `bare/keys/` so `ref_manager.read_ref` returns `None` naturally (file not present).
- `_BlobServer` uses `http.server.HTTPServer` + daemon thread — no mocking; `urlopen` hits a real server.
- All fake API classes inherit from `Vault__API` to satisfy Type_Safe's type-compatibility check on `Vault__Sync__Base.api`.
