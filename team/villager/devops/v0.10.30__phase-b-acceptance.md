## 2. Gate decision: **PASS**

**Rationale.** Every hard correctness metric passes unconditionally:
2,105 / 2,105 tests green, zero failures or skips, coverage held at 86%
(85.638% → 85.638%), mock count unchanged at 553, no `__init__.py`
introduced under `tests/`, and `git diff origin/dev -- sgit_ai/` is
empty. The headline runtime gate (≤ 80s combined CI parallel) lands at
**71.29s** on this runner — comfortably under the 80s gate, with the
parallel pass alone at 67.98s and the marker pass at 3.31s (no tests
collected, exit 5 handled). The relative speedup tells the same story:
parallel wall clock is **29.3%** of serial wall clock on this same
runner (3.4× speed-up on 4 vCPUs), in line with brief 05's projection.
The three rows that fail their absolute targets (#3 tests/sec serial,
#13 slowest file, #14 slowest test) are all runtime metrics that scale
with host CPU speed; this runner is ~1.81× slower than the
brief-01/02 baseline host, so a 16.9 → 9.4 tests/sec dilation, an 11.70
→ 20.15s slowest-file dilation, and a 3.29 → 6.42s slowest-test
dilation are exactly the expected runner-speed shadow, not Phase B
regressions. The relative improvement vs pre-Phase-B on this same
runner (224s vs 243.11s warm serial, −7.8%) confirms the brief-04
fixture savings landed.

## 5. Comparison narrative — where the wins came from

**Brief 04 (shared fixtures) — −7.8% warm serial on this runner.**
The 4 refactored files (per brief 04 §8) showed a consolidated saving
of 18.33s (57.44s → 39.11s) on this same sandbox. This sign-off run
re-measured the same 4 files together at **37.62s**, consistent with
brief 04's 39.11s post-refactor figure (and well below the 57.44s
pre-refactor sum). Top contributor: `test_Vault__Bare.py` (9.51s →
1.15s, −8.36s) by replacing per-test full vault construction with the
F4 `bare_vault_workspace` factory. Probe (−4.57s) and Simple_Token
(−4.57s) saved through F5 session-scoped `Vault__Test_Env` and the F6
post-push origin snapshot for the 2 share-safe clone tests. PKI
saving was modest (−0.83s) because Sign_Verify and Encrypt_Decrypt
were already snapshot-optimised pre-brief — F1/F2 only replaced the
RSA-4096 generation in Export_Delete and Import_Contacts.

**Brief 05 (parallelization) — the headline gate hit.** The suite
proved parallel-safe out of the box (zero `no_parallel` markers
needed; per-test `tempfile.mkdtemp()` already isolates state), so
`-n auto` cleanly distributes across 4 vCPUs. On this run the parallel
pass landed at **67.98s** (re-run 67.50s — stable), under the 80s gate
even before adding the negligible 3.31s serial-marker pass. That gives
a measured **3.4× speedup** on a 4-vCPU host — within the design's
3.2× projection. The two-pass CI invocation in
`.github/actions/pytest__run-tests/action.yml` is in place; coverage
stayed at 86% under xdist (no line-coverage regression).

**Residual gaps.** (a) The ≤ 115s warm-serial design-calibrated target
(#1) is unreachable on this runner without a faster host — at 224s
this runner is 1.81× the brief-02 baseline, so the comparable target
on this host would be ~208s, which the actual 224s is close to but
not under. The relative −7.8% improvement is intact and matches what
brief 04 §8 measured. (b) The ≤ 60s stretch parallel target is
unreachable on a 4-vCPU host (brief 05 documented this); GitHub
runners with higher vCPU counts could close the gap. (c) #13 and #14
absolute targets (≤ 9s file, ≤ 2.5s test) shadow the host slowdown;
brief 04 explicitly carved out the slowest hot tests (e.g. the 8/10
fresh-state tests in `test_Vault__Sync__Simple_Token.py`) as
correctness-preserving non-share-safe, accepting their cost. None of
these residuals block the deferred queue.
