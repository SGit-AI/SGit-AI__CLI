## What Still Needs To Be Done (Ordered by Priority)

### HIGH — Blocks real-world usage

1. **Wire backends into sync** — `push()` and `pull()` should use `Vault__Backend` instead of calling `Vault__API` directly. This is what makes the backend abstraction actually useful.
2. **Auto-drain GC** — Add `self.gc_drain(directory)` calls at the start of `push()` and `pull()`.
3. **Integration test against dev server** — The batch endpoint is live at `dev.send.sgraph.ai` (`sgraph-ai-app-send` v0.15.0). Need to verify request/response format compatibility with an end-to-end test.

### MEDIUM — Needed for complete feature set

4. **Signature verification on GC drain** — Verify the change pack's ECDSA signature before integrating blobs.
5. **`--branch-only` push** — Push clone branch ref + objects without touching named branch (Phase D.5).
6. **Zip backend** — `Vault__Backend__Zip` for portable vault export/import (Phase E.2).
7. **Double-encrypted named branch private key** — `ENC(user_pub_key, ENC(read_key, private_key))` for locked branches (Phase D.3).

### LOW — Polish / deferred

8. **Health check records** (Phase F.5) — Periodic integrity validation.
9. **Scoped write-only tokens** (Phase F.2) — Server-side auth, not CLI.
